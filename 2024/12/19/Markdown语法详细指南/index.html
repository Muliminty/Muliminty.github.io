Markdown 语法详细指南

Markdown 是一种轻量级标记语言，用于格式化文本。它使用简单的语法来创建格式化的文档，易于阅读和编写。

标题

Markdown 支持 6 级标题，使用 # 符号表示：

# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题

效果：

一级标题

二级标题

三级标题

四级标题

五级标题
六级标题

注意：

  • # 后面需要加一个空格
  • 也可以使用下划线语法（仅支持一级和二级）：
    一级标题
=========

段落和换行

段落

段落之间使用一个空行分隔：

这是第一个段落。

这是第二个段落。

换行

在行末添加两个空格，然后按回车键可以创建换行：

这是第一行  
这是第二行

或者使用 HTML 的 <br> 标签：

这是第一行<br>
这是第二行

文本格式

粗体

使用 ** 或 __ 包裹文本：

**粗体文本**
__粗体文本__

效果： 粗体文本

斜体

使用 * 或 _ 包裹文本：

*斜体文本*
_斜体文本_

效果： 斜体文本

粗斜体

使用 *** 或 ___ 包裹文本：

***粗斜体文本***
___粗斜体文本___

效果： 粗斜体文本

删除线

使用 ~~ 包裹文本：

~~删除的文本~~

效果： 删除的文本

高亮

使用 == 包裹文本（部分 Markdown 解析器支持）：

==高亮文本==

下划线

使用 HTML 标签：

<u>下划线文本</u>

效果： 下划线文本

列表

无序列表

使用 -、* 或 + 创建无序列表：

- 项目 1
- 项目 2
  - 子项目 2.1
  - 子项目 2.2
- 项目 3

效果：

  • 项目 1
  • 项目 2
    • 子项目 2.1
    • 子项目 2.2
  • 项目 3

有序列表

使用数字加 . 创建有序列表：

1. 第一项
2. 第二项
3. 第三项
   1. 子项 3.1
   2. 子项 3.2

效果：

  1. 第一项
  2. 第二项
  3. 第三项
    1. 子项 3.1
    2. 子项 3.2

注意： 数字可以不连续，Markdown 会自动编号。

任务列表

使用 - [ ] 或 - [x] 创建任务列表：

- [ ] 未完成的任务
- [x] 已完成的任务
- [ ] 另一个未完成的任务

效果：

  • 未完成的任务
  • 已完成的任务
  • 另一个未完成的任务

链接

行内链接

[链接文本](https://example.com)
[链接文本](https://example.com "提示文本")

效果： 链接文本

引用式链接

[链接文本][链接标识]

[链接标识]: https://example.com "可选的提示文本"

效果： 链接文本

自动链接

直接使用 URL：

https://example.com
<https://example.com>
<email@example.com>

效果：

https://example.com
https://example.com
email@example.com

锚点链接

链接到文档内的标题：

[跳转到标题](#标题)

图片

行内图片

![图片描述](图片URL)
![图片描述](图片URL "可选标题")

引用式图片

![图片描述][图片标识]

[图片标识]: 图片URL "可选标题"

带链接的图片

[![图片描述](图片URL)](链接URL)

示例：

![示例图片](https://via.placeholder.com/150 "占位图片")

本地图片

在 Hexo 中使用本地图片，需要将图片放在与文章同名的文件夹中：

{% asset_img test-image-1.png "本地图片标题" 本地图片示例 %}

效果：

重要提示：

  1. 图片路径：必须使用文章标题作为路径前缀

    • 文章名：Markdown语法详细指南.md
    • 图片路径：Markdown语法详细指南/图片名.png

  2. 图片文件夹：创建文章时会自动创建同名文件夹（如果 post_asset_folder: true）

  3. 引用方式：

    ![图片描述](文章标题/图片文件名.png)

  4. 多个图片示例：

{% asset_img test-image-1.png "第一张测试图片" 测试图片1 %}

{% asset_img test-image-2.png "第二张测试图片" 测试图片2 %}

效果：

代码

行内代码

使用反引号 ` 包裹代码：

这是 `行内代码` 的示例。

效果： 这是 行内代码 的示例。

代码块

使用三个反引号创建代码块：

```语言名称
代码内容
```

示例：

function hello() {
  console.log("Hello, World!");
}
def hello():
    print("Hello, World!")
echo "Hello, World!"

在代码块开头指定语言名称即可启用语法高亮：

```javascript
const name = "Markdown";
console.log(name);
```

  • python 或 py
  • html
  • css
  • bash 或 sh
  • json
  • yaml 或 yml
  • markdown 或 md
  • java
  • go
  • rust
  • cpp 或 c++

表格

使用 | 和 - 创建表格：

| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容1 | 内容2 | 内容3 |
| 内容4 | 内容5 | 内容6 |

效果：

列1 列2 列3
内容1 内容2 内容3
内容4 内容5 内容6

对齐方式

使用 : 控制对齐方式：

| 左对齐 | 居中对齐 | 右对齐 |
|:-------|:--------:|-------:|
| 内容1  | 内容2    | 内容3  |
| 内容4  | 内容5    | 内容6  |

效果：

左对齐 居中对齐 右对齐
内容1 内容2 内容3
内容4 内容5 内容6

表格中的特殊字符

在表格中使用 | 需要转义：

| 列1 | 列2 |
|-----|-----|
| 内容 \| 更多内容 | 其他内容 |

转义字符

  • [] 方括号

脚注

使用 [^标识] 创建脚注：

这是正文内容[^1]，还有另一个脚注[^note]。

[^1]: 这是第一个脚注的内容。
[^note]: 这是第二个脚注的内容。

效果：

这是正文内容^1，还有另一个脚注^note。

数学公式

行内公式

使用 $ 包裹：

这是行内公式：$E = mc^2$

效果： 这是行内公式：$E = mc^2$

块级公式

使用 $$ 包裹：

$$
\int_{-\infty}^{\infty} e^{-x^2} dx = \sqrt{\pi}
$$

效果：

$$
\int_{-\infty}^{\infty} e^{-x^2} dx = \sqrt{\pi}

注意： 数学公式需要支持 LaTeX 的 Markdown 解析器（如 Hexo 的某些主题）。

最佳实践

1. 标题层级

  • 从一级标题开始，按顺序使用
  • 不要跳过标题层级
  • 保持标题层级的一致性

2. 列表

  • 使用一致的列表符号（建议使用 -）
  • 保持缩进的一致性
  • 列表项之间使用空行分隔（可选）

3. 代码块

  • 始终指定语言名称以获得语法高亮
  • 保持代码的缩进和格式
  • 对于长代码，考虑使用引用式链接

4. 链接和图片

  • 使用描述性的链接文本
  • 为图片添加有意义的 alt 文本
  • 考虑使用引用式链接以提高可读性

5. 表格

  • 保持表格对齐
  • 使用表头行
  • 对于复杂表格，考虑使用 HTML

6. 转义

  • 当需要显示特殊字符时，使用转义字符
  • 在代码块中不需要转义

7. 兼容性

  • 不同 Markdown 解析器可能有差异
  • 测试你的 Markdown 在不同平台上的显示效果
  • 某些扩展语法可能不被所有解析器支持

总结

Markdown 是一种简单而强大的标记语言，掌握这些基本语法就能创建格式良好的文档。记住：

  1. 简洁性：Markdown 的语法简单直观
  2. 可读性：原始 Markdown 文本也易于阅读
  3. 兼容性：大多数平台都支持 Markdown
  4. 扩展性：许多平台支持扩展语法（如表格、任务列表等）

开始使用 Markdown，让文档编写变得更加高效！
