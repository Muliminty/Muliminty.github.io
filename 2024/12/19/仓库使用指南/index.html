MulimintyBlog 仓库使用指南

本文档介绍如何使用和配置这个 Hexo 博客仓库。

项目概述

这是一个基于 Hexo 6.3.0 构建的静态博客项目，使用 minima 主题，支持通过 Git 部署到 GitHub Pages。

环境要求

在开始之前，请确保已安装以下软件：

  • Node.js: 版本 10.13 或更高（建议使用 Node.js 12.0 及以上版本）
  • Git: 用于版本控制和部署
  • npm: 通常随 Node.js 一起安装

安装步骤

1. 克隆仓库

git clone <repository-url>
cd MulimintyBlog

2. 安装依赖

npm install

这将安装所有必需的 Hexo 插件和主题依赖。

项目结构

MulimintyBlog/
├── _config.yml              # Hexo 主配置文件
├── package.json             # 项目依赖配置
├── source/                  # 源文件目录
│   ├── _posts/             # 博客文章目录
│   │   ├── hello-world.md
│   │   └── hexo使用笔记.md
│   └── about/              # 关于页面
├── themes/                  # 主题目录
│   ├── cactus/            # Cactus 主题
│   ├── minima/            # Minima 主题（当前使用）
│   └── tranquility/       # Tranquility 主题
└── scaffolds/              # 模板文件

常用命令

基础命令

# 创建新文章
hexo new "文章标题"
# 或使用缩写
hexo n "文章标题"

# 创建新页面
hexo new page "页面名称"

# 生成静态文件
hexo generate
# 或使用缩写
hexo g

# 启动本地服务器（默认端口 4000）
hexo server
# 或使用缩写
hexo s

# 部署到远程仓库
hexo deploy
# 或使用缩写
hexo d

# 清理缓存和生成的文件
hexo clean

# 查看帮助
hexo help

# 查看版本
hexo version

组合命令

# 生成并启动本地预览
hexo s -g

# 生成并部署
hexo d -g

# 生成并部署（另一种写法）
hexo g -d

npm 脚本

项目在 package.json 中配置了便捷脚本：

# 启动开发服务器
npm run server

# 生成静态文件
npm run build

# 清理生成的文件
npm run clean

# 部署到 GitHub Pages
npm run deploy

配置说明

主配置文件 (_config.yml)

主要配置项包括：

  • 站点信息: 标题、作者、语言等
  • URL 设置: 站点地址和永久链接格式
  • 主题设置: 当前使用 minima 主题
  • 代码高亮: 使用 Prism.js 进行代码高亮
  • 部署配置: Git 部署到 GitHub Pages

重要配置项

1. 站点信息

title: Muliminty's Homepage
author: Muliminty
language: zh-CN

2. 文章资源文件夹

post_asset_folder: true

启用后，创建新文章时会自动创建同名文件夹，用于存放文章相关的图片等资源。

3. 代码高亮

highlight:
  enable: false
prismjs:
  enable: true
  line_number: true

项目使用 Prism.js 进行代码高亮，已禁用 Hexo 默认的高亮功能。

4. 部署配置

deploy:
  type: git
  repo: https://github.com/Muliminty/Muliminty.github.io
  branch: main

⚠️ 安全提示: 如果使用 token 进行部署，建议使用环境变量或 Git 凭据助手，不要将 token 直接写在配置文件中。

写作指南

创建新文章

hexo new "文章标题"

创建后会在 source/_posts/ 目录下生成：

  • 文章标题.md - 文章文件
  • 文章标题/ - 同名文件夹（用于存放图片等资源）

文章 Front Matter

每篇文章开头需要包含 Front Matter：

---
title: 文章标题
date: 2024-12-19 00:00:00
tags:
- 标签1
- 标签2
categories: 分类名称
---

插入图片

配置图片资源文件夹

第一步：修改项目根目录下的 _config.yml 文件，设置 post_asset_folder 为 true：

post_asset_folder: true

第二步：安装 hexo-asset-img 插件：

npm install hexo-asset-img --save

使用图片

完成上述配置后，使用 hexo new "文章标题" 创建新文章时，会在 source/_posts/ 目录下自动创建：

  • 文章标题.md - 文章文件
  • 文章标题/ - 同名文件夹（用于存放图片等资源）

图片引用方式：

![图片描述](文章标题/图片文件名.png)

重要提示：

  • 引用图片时一定要带上文章标题作为路径前缀
  • 图片文件需要放在与文章同名的文件夹中
  • 例如：文章名为 hexo使用笔记.md，图片应放在 hexo使用笔记/ 文件夹中，引用时使用 hexo使用笔记/图片名.png

示例：

创建文章后，目录结构如下：

source/_posts/
├── 文章标题.md
└── 文章标题/
    ├── 图片1.png
    └── 图片2.jpg

在文章中使用图片：

![图片描述](文章标题/图片1.png)

主题使用

当前主题：Minima

项目配置使用 minima 主题。如果需要切换主题：

  1. 在 _config.yml 中修改 theme 配置
  2. 确保主题文件存在于 themes/ 目录下
  3. 重新生成静态文件

可用主题

  • minima: 当前使用的主题
  • cactus: 备选主题（已安装）
  • tranquility: 备选主题（已安装）

部署流程

部署到 GitHub Pages

  1. 配置部署信息

    确保 _config.yml 中的部署配置正确：

    deploy:
  type: git
  repo: <your-repo-url>
  branch: main

  2. 生成并部署

    hexo clean
hexo generate
hexo deploy

    或使用组合命令：

    hexo clean && hexo g -d

  3. 验证部署

    访问 https://<username>.github.io 查看部署结果。

插件说明

项目使用了以下主要插件：

  • hexo-asset-img: 图片资源处理
  • hexo-deployer-git: Git 部署支持
  • hexo-generator-archive: 归档页面生成
  • hexo-generator-category: 分类页面生成
  • hexo-generator-tag: 标签页面生成
  • hexo-generator-index: 首页生成

常见问题

1. 本地预览无法访问

  • 检查端口 4000 是否被占用
  • 尝试使用其他端口：hexo server -p 5000

2. 部署失败

  • 检查 Git 配置是否正确
  • 确认部署仓库地址和分支名称
  • 检查是否有部署权限

3. 图片无法显示

  • 确认图片路径是否正确
  • 检查是否使用了文章标题作为路径前缀
  • 确认图片文件是否存在于同名文件夹中

4. 代码高亮不生效

  • 确认 Prism.js 已启用
  • 检查代码块语法是否正确
  • 查看主题是否支持 Prism.js

开发工作流

推荐的开发流程：

  1. 创建文章: hexo new "文章标题"
  2. 本地预览: hexo server 或 npm run server
  3. 编辑内容: 在编辑器中修改文章
  4. 实时预览: 浏览器中查看效果
  5. 生成部署: hexo g -d 或 npm run deploy

更新和维护

更新 Hexo

npm update hexo

更新依赖

npm update

清理缓存

如果遇到奇怪的问题，尝试清理：

hexo clean
rm -rf node_modules
npm install

参考资源

  • Hexo 官方文档
  • Hexo 配置文档
  • Minima 主题文档

总结

这个仓库提供了一个完整的 Hexo 博客解决方案，包含了常用的插件和主题配置。通过本文档，你应该能够：

  • ✅ 了解项目结构
  • ✅ 掌握常用命令
  • ✅ 创建和编辑文章
  • ✅ 部署到 GitHub Pages
  • ✅ 解决常见问题

如有其他问题，请参考 Hexo 官方文档或提交 Issue。
